# 桌面 MVP 技术选型

本文档记录桌面 MVP 的轻量技术判断。当前原则是功能优先，先验证透明桌宠体验和外部 AI 事件桥，不追求完整工程体系。

## 候选方案

### Tauri

优点：

- 运行时轻，适合长期挂在桌面的宠物应用。
- 支持透明窗口、置顶、托盘、文件系统访问和本地 HTTP。
- 前端可以复用当前 Web POC 的 React 渲染经验。

风险：

- 透明窗口、点击穿透、多显示器和 macOS 权限细节需要逐项验证。
- 需要 Rust/Tauri 打包链路。

### Electron

优点：

- 桌面能力成熟，透明窗口、托盘、IPC 和生态资料多。
- 对纯前端工程更友好。

风险：

- 资源占用更高。
- 如果只是桌宠和本地事件桥，运行时偏重。

## 当前建议

MVP 先选 Tauri。

原因：

- 产品形态是长期驻留桌面的小应用，低资源占用更重要。
- 当前渲染已经是 Web 技术，迁移到 Tauri 前端成本较低。
- 本地 HTTP 事件桥、宠物包导入、设置持久化都能在 Tauri 能力范围内验证。

如果 Tauri 在透明窗口或点击穿透上遇到平台阻碍，再保留 Electron 作为备选。

## MVP 验证清单

1. 透明无边框窗口显示宠物。
2. 窗口置顶，可拖拽移动。
3. 宠物包从本地目录导入。
4. 复用 AI Pet Protocol 解析和渲染规则。
5. 本地 HTTP 接收外部 AI 事件。
6. 非循环动作结束后回 idle。
7. 托盘菜单提供退出、切换宠物、打开设置。
8. Windows 和 macOS 各跑通一次开发构建。
