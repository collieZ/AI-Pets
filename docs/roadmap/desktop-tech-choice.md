# 桌面 MVP 技术选型

本文档记录桌面 MVP 的轻量技术判断。当前原则是功能优先，先验证透明桌宠体验和外部 AI 事件桥，不追求完整工程体系。

## 当前方案：Electron

优点：

- 桌面能力成熟，透明窗口、置顶、无边框窗口、托盘、IPC 和本地 HTTP 都有成熟路径。
- 对当前纯前端工程更友好，不需要先安装 Rust/Cargo。
- 更容易快速跑通 Windows/macOS 的第一版桌面壳。
- 自带 Chromium，渲染一致性更强，前期排查 UI 问题更省心。

风险：

- 运行时和安装包更大，常驻桌面时资源占用需要后续优化。
- 长期产品化时需要关注内存、启动速度和自动更新体积。

## 选择结论

MVP 选 Electron。

原因：

- 当前阶段功能优先，目标是尽快看到可运行的桌面宠物。
- Electron 不依赖 Rust 环境，能直接基于现有 Node/React/Vite 工作流推进。
- Electron 的透明无边框窗口、置顶、托盘和 IPC 资料更多，适合快速验证桌宠体验。

## MVP 验证清单

1. 透明无边框窗口显示宠物。
2. 窗口置顶，可拖拽移动。
3. 宠物包从本地目录导入。
4. 复用 AI Pet Protocol 解析和渲染规则。
5. 本地 HTTP 接收外部 AI 事件。
6. 非循环动作结束后回 idle。
7. 托盘菜单提供退出、切换宠物、打开设置。
8. Windows 和 macOS 各跑通一次开发构建。
