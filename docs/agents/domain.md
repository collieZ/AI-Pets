# Domain Docs

本文说明工程类 skills 在探索代码前应如何读取本仓库的领域文档。

## 探索前优先阅读

- 根目录 **`CONTEXT.md`**。
- **`docs/adr/`** 中与当前修改区域相关的架构决策记录。

如果这些文件不存在，继续工作即可，不要因为文件缺失而阻塞，也不要主动要求先创建。领域建模类 skill 会在术语或决策真正沉淀时再懒创建。

## 文件结构

本仓库使用 single-context 布局：

```text
/
├── CONTEXT.md
├── docs/adr/
└── apps/
    ├── desktop/
    └── web-poc/
```

## 使用 glossary 中的词

输出中提到领域概念时，优先使用 `CONTEXT.md` 中定义的词。这适用于 issue 标题、重构建议、bug 假设、测试名称和架构说明。

如果需要的概念还没有进入 glossary，要么避免发明新词，要么把这个空缺记录给后续领域建模。

## 标出 ADR 冲突

如果输出内容与已有 ADR 冲突，必须明确指出，而不是静默覆盖：

> 与 ADR-0007 (...) 冲突，但可能值得重新打开，因为...
